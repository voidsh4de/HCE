Window

We recommend the game is contained in a function called start().

defining the window size can easily be achived by using the following method:

engine._widthPX = 1000;
engine._heightPX = 600;

the values are in PIXELS and can be set to any positive value you wish.

defining the window size can easily be achived by using the following method:

engine.setDimensions(width, height)

the values are in PIXELS and can be set to any positive value you wish.

after the width and height have been defined you can declare and establish the window and engine requirements via the following method

engine.makeWindow()

You can also customize the color with the following method: (any css coloring method is supported)

engine.window.changeColor(COLOURVALUE);

A image can be used as a background in place of a solid colour via the following function:

engine.window.backgroundIMG(URL)

Buttons

there are two types of button input within the HyperCascadedEngine, UI and keyboard.
To Create a UI button you can use the following function

engine.buttons.makeButton(ACTION, NAME);

to use direct keyboard input you must first search the keyboard, then you can check the information collected by your search.

engine.buttons.checkKeys(); //updates the keys object
if (engine.buttons.keys["w"]) x += 5; //"w" and [x += 5] are fillers for the example

you can delete/clear all UI type buttons with the following function:

engine.buttons.clearButtons

items

items are the class of which anything displayed in your game is under, whether it be the player or a chair, it is an item.
they are also the most complicated part of the engine. first lets cover how to make a item, a item can be made with the following function:

engine.items.makeItem(XPOSITION, YPOSITION, NAME, WIDTH, HEIGHT, COLOUR, RADIUS, BORDER);

XPOSITION and YPOSITION refer to pixels, the bottom left of your window is (0,0), you dont define that its pixels you just set the value to a num, that also applys to RADIUS. RADIUS and BORDER refer to the cascaded styling sheets style functions, RADIUS and BORDER are also optional inputs.


heres all the items sub varibles

item._x; //refers to XPOSITION
item._y; //refers to YPOSITION
item.name;
item._width; //width of item
item._height; //height of item
item.color; //colour of item
item.radius; //css style radius of item
item.border; //css style border "black solid 1px"
item.itemOffset; //the X offset that is applied to this item on print

to display your item(s) to the window you can use the following function

engine.items.printItem(ITEM, IMAGE);

the image URL must be wraped in "url(URL)".

you must do this function for every item you wish to be displayed. IMAGE is not required and is used when you have art for a item instead of a solid colour.

To clear/refresh the window you can use the following function:

engine.items.clearItems();

if you intend to delete a item you MUST use the following function to stop positoning issues

engine.items.removeOffset(ITEMWIDTH);

You can detect collision via the following function: (it returns a bool value)

engine.items.isCollision(ITEM, ITEM); //fairly simple

Sound

sound is very simple with HyperCascadedEngine, you can save a sound and use them using the following methods:

const sound = engine.sound.make(FILE); //save this to a const
sound.play();
sound.pause();
